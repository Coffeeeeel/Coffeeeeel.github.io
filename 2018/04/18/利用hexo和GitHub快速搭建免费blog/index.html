利用hexo+GitHub快速搭建免费blog

2018-04-18

参考了一些文章，最终决定利用hexo这个博客框架加GitHub搭建自己的博客，目的也是为了督促自己能够多写一些东西。该文主要参考了@无小意博文中的路线，因此基本上没遇到什么坑，此外还用了@yanm1ng配置的vexo主题（简洁漂亮），特此感谢。

零 环境配置

  • windows10 64位系统
  • Notepad编辑器
  • Node.js安装
  • Git安装及配置
  • hexo博客框架（cmd中输入npm install -g hexo-cli 警告可以忽略）
    • hexo文档
    • vexo主题
  • 有道云笔记（支持markdown编辑）

一 GitHub配置

参考@廖雪峰老师关于Git的文章进行配置，然后新建一个repository，并按此以下方式进行命名：yourname.github.io，其中yourname为你注册名，该地址就是你的博客域名。

二 博客的建立

  1. 自己在某盘创建一个文件夹（比如test），cmd内cd进入test文件夹
  2. 在文件夹内进入Git hash 输入hexo init blog（此处如果出现警告可忽略）
  3. 提示Start blogging with Hexo，就是安装成功了
  4. 文件夹中自带一篇文章“Hello World”
  5. 命令行进入blog目录下
  6. 输入hexo g，生存静态文件
  7. 输入hexo s，启动服务器。默认情况下，访问网址为： http://localhost:4000/
    • 此处若无法打开可能是端口被占用，可用命令hexo s -p 4002或其他端口进行测试
  8. 如可以正常访问则重新打开cmd，输入：ssh-keygen -t rsa -C “Github的注册邮箱地址”，一路Enter得到信息:Your public key has been saved in /c/Users/user/.ssh/id_rsa.pub
    • 找到该文件，将id_rsa.pub里的内容粘贴到github的ssh配置中
    • New SSH key ——Title：blog —— Key：输入刚才复制的—— Add SSH key

如果是跟廖雪峰的git学习资料联系过的，第8个步骤可以略过（配置过了）。

三 博客网站配置

1.在本地test文件中打开_config.yml文件进行参数配置

title: 你的博客名
subtitle: 你的博客的小名？
description: 博客描述，可能有关于搜索引擎抓取
author: 作者
language: 语言配置
timezone: 当地时间

特别提醒一下，每个自定义参数之前都要有个空格，如 title：（空格）你的博客名 ，否则没法识别。

2.部署服务器的相关配置信息，只需要修改repo部分，username设置为你的username

deploy: 
  type: git
  repo: https://github.com/username/username.github.io.git
  branch: master

四 博客发布

好了，搞到现在可以试着发一篇blog了。

  1. 进入test/blog文件夹下，git bash中输入hexo new “Hello blog”
  2. 打开Info中的地址，打开文件
  3. 用markdown语法进行编辑，可以用支持该语法的编辑器进行编辑，比如我用的有道云笔记
  4. 编写测试内容：
  1. 执行以下步骤：
D:\test\blog
$ hexo clean
INFO  Deleted database.
INFO  Deleted public folder.

D:\test\blog
$ hexo generate
INFO  Start processing
INFO  Files loaded in 1.48 s
...
INFO  29 files generated in 4.27 s

D:\test\blog
$ hexo server
INFO  Start processing
INFO  Hexo is running at http://localhost:4000/. Press Ctrl+C to stop.
  1. 打开http://localhost:4000/之后，将你的blog部署到服务器上：
D:\test\blog
$ hexo deploy
INFO  Deploying: git
...

 其中，可能会出现github登录界面，正常填写就行,hexo deploy 后,出现  error deployer not found:git 的错误，输入以下代码:

npm install hexo-deployer-git –save
hexo deploy

“Hello Blog”

五 主题配置

 hexo提供了非常多的主题，大家可以随意尝试，更换主题的步骤如下：

  1. 进入官网主题仓库，选择自己喜欢的主题，进入他的github里下载到本地，或者按照作者的提示进行安装
  2. 如果下载到本地可以将主题文件放在blog/themes文件夹下
  3. 在blog下的_config.yml中下面的theme属性改为你下载的主题文件名
  4. 然后按照作者提示进行自己的配置
  5. 这里特别要注意一点：如果该主题有一些其他的页面，如tags，projection等页面，一定要将主题文件/_source下的相关文件拷贝到blog/source下，否则这些页面找不到
  6. 最后重新输入：
hexo clean
hexo generate
hexo deploy

 好了，现在你可以享受你的博客了。

六 写在最后

 因为有了前人的经验，所以少了很多坑，希望这篇文章也能够帮助到你。在博客搭建的过程中你也可能遇到一些上面没有提到的问题，但是相信你都可以通过搜索找到答案。

  毕竟自己折腾搭建博客的人，都是一群不怕麻烦的神经病吧…

以上，祝好。
